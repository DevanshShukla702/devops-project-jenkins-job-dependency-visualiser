Jenkins Concepts Guide

Everything you need to understand the visualizer.

🏗️ What is Jenkins?

Jenkins is the world's leading open-source automation server. It provides hundreds of plugins to support building, deploying, and automating any project. Jenkins helps development teams reliably build software through Continuous Integration and Continuous Delivery (CI/CD) pipelines.

📦 What is a Jenkins Job?

A "Job" (or Project) in Jenkins is a user-configured description of work which Jenkins should perform, such as building a piece of software, running test suites, or deploying an application to a production server.

🔗 What are Job Dependencies?

In complex architectures, jobs rely on one another. An Upstream job is a job that runs before the current job. A Downstream job is a job that is triggered after the current job finishes.

FlowTrace visualizes this flow automatically using vis-network.

🚦 Understanding Pipeline Status

Status Color Code Meaning
SUCCESS Blue / Green The job built successfully.
FAILED Red The execution had fatal errors.
UNSTABLE Yellow Tests failed, but the build compiled.
NOT BUILT Gray Pending, queued, or aborted.

📖 Glossary

Build A single execution of a Jenkins job.
Trigger The event that causes a job to start (e.g., git push, cron schedule, or upstream job finishing).
Node / Edge (Graph) In FlowTrace, a Node represents a Jenkins Job, and an Edge (the connecting line) represents a dependency trigger.
